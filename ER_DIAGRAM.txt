USERS:
  Purpose: Store user account information
  Primary Key: id
  Unique Constraint: email
  Relationships: Has many cart_items, Has many orders

CART_ITEMS:
  Purpose: Store items in user's shopping cart
  Primary Key: id
  Foreign Key: user_id â†’ users(id)
  Relationships: Belongs to one user

ORDERS:
  Purpose: Store order information
  Primary Key: id
  Foreign Key: user_id â†’ users(id)
  Relationships: Belongs to one user, Has many order_items

ORDER_ITEMS:
  Purpose: Store items in each order
  Primary Key: id
  Foreign Key: order_id â†’ orders(id)
  Relationships: Belongs to one order

OTPS:
  Purpose: Store OTP codes for email verification
  Primary Key: id
  No Foreign Keys: Independent entity
  Relationships: None (temporary data)

1. Users:
   - email must be unique
   - password_hash cannot be null
   - is_verified defaults to FALSE

2. Cart_Items:
   - user_id must exist in users table
   - quantity must be > 0
   - item_price must be > 0

3. Orders:
   - user_id must exist in users table
   - status: 'Pending', 'Confirmed', 'Delivered', 'Cancelled'
   - total_amount must be > 0

4. Order_Items:
   - order_id must exist in orders table
   - quantity must be > 0
   - item_price must be > 0

5. OTPs:
   - expires_at must be > created_at
   - otp_code is 6 digits
   - is_used defaults to FALSE

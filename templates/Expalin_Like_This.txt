STEP 1: Database Dikhao (2 min)
─────────────────────────────────
Kaha: phpMyAdmin → http://localhost/phpmyadmin

Kya Bologe:
"Ma'am, yeh MySQL database hai. Mera project mein 5 tables hain."

Dikhaoge:
- restaurant_db (database name)
- 5 tables: users, cart_items, orders, order_items, otps


STEP 2: Tables Explain Karo (3 min)
────────────────────────────────────
Kaha: phpMyAdmin → Click each table

users table:
  - id (Primary Key - unique number)
  - name, email, password
  - Har user ka data

cart_items table:
  - user_id (Foreign Key - users table se link)
  - item_name, price, quantity
  - Shopping cart

orders table:
  - user_id (Foreign Key - users se link)
  - total_amount, status
  - Order history

order_items table:
  - order_id (Foreign Key - orders se link)
  - item_name, price, quantity
  - Order details

otps table:
  - email, otp_code
  - Email verification


STEP 3: Relationships Dikhao (2 min)
────────────────────────────────────
Kya Bologe:
"Ma'am, tables ko Foreign Keys se link kiya hai."

Diagram:
  users (1) ──→ (many) cart_items
  users (1) ──→ (many) orders
  orders (1) ──→ (many) order_items

Matlab:
- Ek user ke multiple cart items
- Ek user multiple orders
- Ek order mein multiple items

STEP 5: SQL Queries (5 min)
───────────────────────────
Kaha: phpMyAdmin → SQL tab

Query 1: Simple SELECT
  SELECT * FROM users;
  → Sab users dikhata hai

Query 2: JOIN (Most Important!)
  SELECT o.id, u.name, o.total_amount
  FROM orders o
  JOIN users u ON o.user_id = u.id;
  → Orders ko users ke saath join karta hai

Query 3: Aggregate
  SELECT SUM(total_amount) FROM orders;
  → Total revenue calculate karta hai

Query 4: GROUP BY
  SELECT status, COUNT(*) FROM orders GROUP BY status;
  → Orders ko status se group karta hai

STEP 7: DBMS Concepts (3 min)
─────────────────────────────

Primary Key:
  - Har row ko unique identify karta hai
  - Example: users.id

Foreign Key:
  - Do tables ko link karta hai
  - Example: cart_items.user_id → users.id
  - Data integrity maintain karta hai

Normalization:
  - Data redundancy remove karna
  - Mera database 3NF mein hai
  - Matlab: No duplicate data

ACID Properties:
  - Atomicity: Sab kuch hoga ya kuch nahi
  - Consistency: Data always valid
  - Isolation: Concurrent operations safe
  - Durability: Data permanent
